# Implementation Summary Protocol

Update `docs/实现说明.md` after every completed implementation.

## Required sections per entry

1. Date and iteration id
2. Background
3. Changed files
4. Reproducible operation steps
5. Functional capabilities added/changed
6. Verification results
7. Next actions

## Writing constraints

- Use concise, action-oriented language.
- Include runnable commands when possible.
- Explicitly mark test status: passed/failed/not-run with reason.
