## 2026-03-05 | V1-002 看板可用性修复（API不可达容错）

### 背景
- 访问 `http://127.0.0.1:8501` 时出现看板不可用。
- 原因是看板启动后会立即请求 API，若 API 未启动或未就绪会直接报错。

### 改动文件
- `apps/web/dashboard.py`

### 操作步骤
1. 在 dashboard 中增加 `safe_get` / `safe_post` 包装。
2. 页面初始化先检查 `/health`。
3. API 不可达时展示错误提示，不再导致页面崩溃。
4. 按钮和数据区域均改为容错输出。

### 功能说明
- 看板现在支持 API 不可达时的友好提示。
- 可先开看板，再启动 API，页面可继续使用。

### 验证结果
- 本地确认 API 与 8501 端口可监听。
- 单元测试通过：`3 passed`。

### 后续事项
- 可增加“重试连接 API”按钮与自动轮询刷新。
